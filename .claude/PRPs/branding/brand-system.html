Button Principles
Primary — frosted ice tint. Used once per view, max. The main action.
Secondary — border only. Everything else. "Open All", "Stop", "Cancel".
Ghost — invisible until hover. Toolbars, menus, icon actions.
Danger — looks secondary. Reveals ember on hover. Quiet until you commit.

Design Principles

The philosophy behind KILD's visual identity.

Cold Precision

Nordic ice meets Japanese blade. The UI should feel sharp, precise, and cold. No soft edges, no warm colors by default. Warmth is reserved for warnings.

Lift the Fog

Every element should increase visibility. Status is clear at a glance. Scale gracefully to 30+ kilds. The Tōryō must see everything.

Keyboard First

Power users live in the keyboard. Every action has a shortcut. The UI should be fully operable without a mouse.

Honest Feedback

No silent failures. Errors are visible. Status is truthful. If something is crashed, it says crashed. The UI doesn't hide problems.
